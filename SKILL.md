# 内容链接速读助手

双引擎速读：秘塔全文抓取 + 小宿智能总结 → AI 结构化摘要。

## MCP 服务配置

本技能依赖以下 MCP 服务，运行时**必须**先读取 `mcp-config.json` 获取服务 URL：

| 环境变量名 | MCP 服务 | mcpId | 用途 |
|------------|----------|-------|------|
| `$MCP_METASO_URL` | 秘塔AI-链接速读 | 9400 | 全文抓取，返回完整网页内容（markdown格式） |
| `$MCP_XIAOSU_URL` | 小宿-链接速读 | 9394 | 智能总结，返回网页内容摘要（markdown格式） |

### 双引擎分工

| 引擎 | 特点 | 适用场景 |
|------|------|----------|
| 秘塔AI | **全文抓取**，返回原始完整内容 | 需要精读全文、提取细节、深度分析 |
| 小宿 | **智能总结**，返回处理后的摘要 | 快速了解大意、时间紧迫、链接很长 |

### 配置读取方式

**约束：调用方 agent 必须在执行任何 MCP 调用前，先读取 mcp-config.json 文件**

```bash
python3 scripts/call_mcp.py list "$MCP_METASO_URL"
```

## 工作流程

### 第一步：识别链接并判断速读模式

收到用户消息后：

1. 从消息中提取 URL（可能有一个或多个）
2. 判断链接类型和速读模式：

| 用户意图 | 速读模式 | 引擎策略 |
|----------|----------|----------|
| "帮我看看这个链接" | 标准速读 | 秘塔全文 → AI 提炼摘要 |
| "快速告诉我说了什么" | 快速摘要 | 小宿总结 → 直接输出 |
| "帮我详细分析这篇文章" | 深度精读 | 秘塔全文 + 小宿总结 → 双引擎交叉验证 |
| "读一下这几个链接" | 批量速读 | 逐个调用 → 汇总对比 |

默认使用**标准速读**模式。

### 第二步：调用速读引擎

#### 标准速读（默认）

优先用秘塔抓取全文，AI 自行提炼：

```bash
# 秘塔AI — 全文抓取
python3 scripts/call_mcp.py call "$MCP_METASO_URL" reader_html_content \
  --params '{"url": "<目标URL>"}'
```

如果秘塔失败，自动降级到小宿：

```bash
# 小宿 — 智能总结（备用）
python3 scripts/call_mcp.py call "$MCP_XIAOSU_URL" get_html_content \
  --params '{"url": "<目标URL>"}'
```

#### 快速摘要

直接用小宿，省去 AI 二次提炼：

```bash
python3 scripts/call_mcp.py call "$MCP_XIAOSU_URL" get_html_content \
  --params '{"url": "<目标URL>"}'
```

#### 深度精读

双引擎并行，交叉验证：

```bash
# 同时调用两个引擎
python3 scripts/call_mcp.py call "$MCP_METASO_URL" reader_html_content \
  --params '{"url": "<目标URL>"}'

python3 scripts/call_mcp.py call "$MCP_XIAOSU_URL" get_html_content \
  --params '{"url": "<目标URL>"}'
```

将两个引擎的结果合并分析，取长补短。

### 第三步：AI 结构化输出

基于速读引擎返回的内容，AI 提炼为结构化摘要。

## 输出格式

### 标准速读

```
## 速读摘要

**来源**：<网站名称> | <作者（如有）>
**链接**：<原始URL>
**发布时间**：<时间（如有）>

### 一句话摘要
<用一句话概括全文核心内容>

### 核心观点
1. <观点一>
2. <观点二>
3. <观点三>

### 关键信息
- <关键数据/事实/结论>
- <关键数据/事实/结论>

### 关键词
`关键词1` `关键词2` `关键词3` `关键词4` `关键词5`

### 值得注意
<文章中的重要细节、争议点、或容易忽略的信息>
```

### 快速摘要（简洁版）

```
## 速读

**<一句话摘要>**

要点：
- <要点1>
- <要点2>
- <要点3>

关键词：`词1` `词2` `词3`
```

### 深度精读

```
## 深度精读：<文章标题>

**来源**：<网站名称> | <作者>
**链接**：<原始URL>

### 全文概要
<200字以内的全面概括>

### 核心论点与论据

- 论据：<支撑数据/事例>
- 评价：<论据是否充分>

#### 论点二：<论点>
- 论据：<支撑数据/事例>
- 评价：<论据是否充分>

### 文章结构分析
- 写作方式：<叙事/论证/对比/...>
- 信息密度：高/中/低
- 立场倾向：客观/偏向XX

### 关键数据
| 数据项 | 数值 | 上下文 |
|--------|------|--------|
| <指标> | <值> | <出现在什么语境> |

### 关键词
`关键词1` `关键词2` `关键词3` `关键词4` `关键词5`

- <基于文章内容引发的思考或追问>
```

```

### 总览

|---|------|----------|--------|

- <各文章之间的不同立场或补充信息>

### 逐篇摘要
（按上方标准速读格式逐篇输出）
```

## 链接类型适配

| 链接类型 | 处理策略 |
|----------|----------|
| 新闻/博客文章 | 标准流程 |
| 微信公众号 | 可能需要多次尝试，部分链接有访问限制 |
| 知乎回答/文章 | 注意区分问题和回答 |
| GitHub README | 提取项目介绍、功能、使用方法 |
| GitHub Issue/PR | 提取问题描述、讨论要点、结论 |
| 论文/PDF链接 | 全文抓取后重点提取摘要、方法、结论 |
| 视频页面 | 提取标题、简介、评论区要点（正文内容有限） |

---

## 注意事项

- 秘塔为主引擎（全文），小宿为辅引擎（总结）+ 备用
- 如果秘塔调用失败，自动降级到小宿，不中断流程
- 如果两个引擎都失败，告知用户链接可能无法访问，建议检查
- 批量速读时，逐个处理，一个失败不影响其余
- 对于需要登录才能访问的链接（如部分公众号），如实告知抓取限制
- 输出摘要时保持客观，如原文有争议观点需标注"文章观点"而非事实
- 关键词提取 3-5 个，优先提取专有名词和核心概念
